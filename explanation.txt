
TRANSFORMER TEXT CLASSIFICATION – EXPLANATION

 PROJECT OVERVIEW
This program implements a simple Transformer-based text classification model using PyTorch.
The goal is to classify news articles into one of three categories:
• Sports
• Technology
• Politics

The implementation demonstrates the core ideas behind Transformers:
Embeddings, Self-Attention, Feed Forward Processing, and Classification.

------------------------------------------------------------
 WHY TRANSFORMERS?
------------------------------------------------------------
Transformers solve the long-range dependency problem using Self-Attention.Instead of processing one word at a time, every word can 
directly interact with every other word.
Example:
animal ↔ dog
animal ↔ saw
animal ↔ yesterday
This allows the model to understand relationships between distant words more effectively.

------------------------------------------------------------
 DATASET LOADING
------------------------------------------------------------

The dataset is stored in: dataset.txt Each line follows:
Label|Text
The code reads:
• Labels
• Text articles
and stores them in separate lists.
------------------------------------------------------------
 TOKENIZATION
------------------------------------------------------------

Function:
tokenize(text)
Process:

1. Convert text to lowercase.
2. Split text into words.

Example:

Input:
"Virat Kohli Scored a Century"

Output:
["virat", "kohli", "scored", "a", "century"]

------------------------------------------------------------
 VOCABULARY CREATION
------------------------------------------------------------

A vocabulary maps each word to a unique number.

Special tokens:

<PAD> = 0
<UNK> = 1

Example:

{
    "<PAD>":0,
    "<UNK>":1,
    "virat":2,
    "kohli":3,
    "century":4
}

Vocabulary size = total unique words.
------------------------------------------------------------
 TEXT ENCODING
------------------------------------------------------------

Words are converted into numerical IDs.

Example:

["virat","kohli","century"]

becomes

[2,3,4]

Unknown words become:

<UNK> = 1

------------------------------------------------------------
LABEL ENCODING
------------------------------------------------------------
Labels are converted to numbers.
Sports      → 0
Technology  → 1
Politics    → 2

Example:
Sports → 0
------------------------------------------------------------
EMBEDDING LAYER
------------------------------------------------------------

Transformer models cannot directly understand word IDs.

Embedding converts each token into a dense vector.

Code:

self.embedding = nn.Embedding(vocab_size, d_model)

In this project:

d_model = 32

Meaning every word becomes a vector of length 32.

Example:

"cat"

might become:

[0.17, -0.31, 0.89, ...]

32 values total.

For a sentence containing 10 words:

Shape:

10 × 32

According to Transformer terminology:

d_model = hidden representation size.

------------------------------------------------------------
SELF-ATTENTION
------------------------------------------------------------

The most important part of the Transformer.

The model creates:

Q = Query
K = Key
V = Value

Using:

self.Wq
self.Wk
self.Wv

Code:

Q = self.Wq(x)
K = self.Wk(x)
V = self.Wv(x)

------------------------------------------------------------
ATTENTION SCORE
------------------------------------------------------------

Attention scores determine how much one word should focus on another.

Formula:
Score = Q × K^T

Code:

scores = torch.matmul(Q, K.transpose(1,2))

Then scaled:

scores = scores / sqrt(d_model)
This prevents very large values.

------------------------------------------------------------
 SOFTMAX
------------------------------------------------------------
Softmax converts scores into probabilities.
Code:
weights = torch.softmax(scores, dim=-1)
Each row sums to 1.
The model learns which words are most important.
------------------------------------------------------------
 ATTENTION OUTPUT
------------------------------------------------------------

The attention weights are applied to V.
Code:
attention_output = torch.matmul(weights, V)

Result:

Each word now contains information gathered from other words in the sentence.

------------------------------------------------------------
FEED FORWARD NETWORK (FFN)
------------------------------------------------------------

In the original Transformer architecture, each token passes through a small neural network after attention.

Purpose:
• Learn richer representations
• Improve feature extraction

Your notes mention:

Token Representation
→ Feed Forward Network
→ Better Representation

This simplified implementation omits the FFN layer to keep the model easy to understand.

------------------------------------------------------------
 RESIDUAL CONNECTIONS
------------------------------------------------------------

Original Transformer paper:
Output = x + F(x)

Instead of:
Output = F(x)
Benefits:
• Easier gradient flow
• Better training stability

------------------------------------------------------------
 LAYER NORMALIZATION
------------------------------------------------------------

After residual addition, Transformers apply LayerNorm.

Formula:

LayerNorm(x + Sublayer(x))

Purpose:
• Stabilize training
• Keep activations within a reasonable range
• Improve convergence speed

our simplified model does not include Layer Normalization.
